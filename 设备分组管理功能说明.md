# 设备管理与分组管理功能实现说明

## 一、数据库设计

### 1. 设备分组表 (device_groups)
- id: 主键
- name: 分组名称
- parent_id: 父级分组ID（支持树形结构）
- user_id: 创建人ID
- created_at/updated_at: 时间戳

### 2. 设备表 (devices)
- id: 主键
- name: 设备名称
- sn: 网关SN编号（唯一）
- status: 状态（online/offline）
- group_id: 所属分组ID（外键）
- address: 所在地址
- creator_id: 创建人ID
- remark: 备注
- iccid: ICCID
- sort: 排序号
- created_at/updated_at: 时间戳

### 3. 测试数据
- 4个测试分组：默认、后纺、前纺、测试分组
- 10条测试设备数据

## 二、后端API实现

### 设备管理接口
1. `GET /api/v1/device/list` - 获取设备列表（支持搜索、状态筛选、分组筛选、分页）
2. `POST /api/v1/device/create` - 创建设备
3. `PUT /api/v1/device/update/{device_id}` - 更新设备
4. `DELETE /api/v1/device/delete/{device_id}` - 删除设备
5. `POST /api/v1/device/batch-update-group` - 批量更新设备分组

### 分组管理接口
1. `GET /api/v1/device/groups/list` - 获取分组列表（树形结构）
2. `POST /api/v1/device/groups/create` - 创建分组
3. `PUT /api/v1/device/groups/update/{group_id}` - 更新分组
4. `DELETE /api/v1/device/groups/delete/{group_id}` - 删除分组
5. `GET /api/v1/device/groups/{group_id}/devices` - 获取分组下的设备
6. `POST /api/v1/device/groups/{group_id}/remove-devices` - 从分组移除设备

## 四、关键特性

### 1. 数据关联
- 设备与分组：多对一关系
- 分组与用户：多对一关系
- 设备与创建人：多对一关系
- 支持未分组设备

### 2. 数据筛选
- 设备列表支持多条件组合筛选
- 分组支持树形展示
- 支持关键词模糊搜索

### 3. 权限控制
- 所有接口都需要登录认证
- 用户只能操作自己创建的数据

### 4. 用户体验
- 所有操作有loading状态
- 操作成功/失败都有消息提示
- 删除操作有二次确认
- 表单验证完善

## 七、注意事项

1. SN编号全局唯一，不可重复
2. 删除有设备的分组会失败
3. 设备SN编号创建后不可修改
4. 所有分页默认每页10条
5. 搜索和筛选可以组合使用
