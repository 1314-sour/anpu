# 密码问题修复说明

## 🔍 问题描述

修改密码后,使用新密码登录显示"密码错误",数据库中的密码哈希与实际不符。

---

## 🐛 问题原因

1. **数据库中的旧密码哈希不正确**
   - 之前可能使用了错误的脚本或SQL插入了不正确的密码哈希
   - 导致即使是正确的密码也无法验证通过

2. **修改密码API可能缺少刷新操作**
   - 虽然不是主要原因,但添加 `db.refresh()` 可以确保对象状态同步

### 2. 优化修改密码API

在 `user_center.py` 的 `change_password` 函数中添加了 `db.refresh(current_user)`:

```python
@router.post("/security/password")
async def change_password(
    password_data: PasswordChange,
    current_user: User = Depends(get_current_user),
    db: Session = Depends(get_db)
):
    """修改密码"""
    # 验证旧密码
    if not verify_password(password_data.old_password, current_user.hashed_password):
        return error_response(400, "旧密码错误")
    
    # 更新密码
    current_user.hashed_password = get_password_hash(password_data.new_password)
    db.commit()
    db.refresh(current_user)  # 刷新对象 ← 新增
    
    return success_response(message="密码修改成功")
```

## 📊 密码哈希说明

### bcrypt密码哈希格式

```
$2b$12$hbo1l13o0m8emyRvJ0tK/uOK5Mlbd1HY4Z2Z1eq9CUA42CdGP1jhO
└─┘ └┘ └──────────────────────────────────────────────────────────┘
 │   │                       │
 │   │                       └─ 哈希值 (31字符)
 │   └─ cost因子 (12 = 2^12次迭代)
 └─ 算法标识符 ($2b$ = bcrypt)

总长度: 60个字符
```

### 为什么每次生成的哈希不同?

bcrypt会为每次哈希生成随机的salt(盐),所以:
- 相同的密码,每次生成的哈希都不同
- 但验证时都能通过
- 这是bcrypt的安全特性

**示例:**
```python
hash1 = get_password_hash('123456')  # $2b$12$abc...
hash2 = get_password_hash('123456')  # $2b$12$def...

# hash1 != hash2
# 但是:
verify_password('123456', hash1)  # True
verify_password('123456', hash2)  # True
```
