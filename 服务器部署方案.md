# 基于 cpolar 的本地前后端公网访问部署方案

本文说明如何将当前 `anpu-frontend`（Vue 2）与 `anpu-backend`（FastAPI）通过 cpolar 外网穿透映射到公网，实现“在浏览器输入公网地址即可访问”。整个过程无需改动前后端源码。

---

## 1. 前置准备

| 类型 | 说明 |
| --- | --- |
| 操作系统 | 64 位 Windows / macOS / Linux 均可，保证能同时运行 Node.js 与 Python 环境。 |
| 开发环境 | Node.js 16.x、npm、Python 3.10+、pip、virtualenv（可选）、Git。 |
| 账号与软件 | cpolar 账号（https://www.cpolar.com/），已登录获取 Authtoken；安装 cpolar 客户端；浏览器可访问 cpolar 内网版面 `http://127.0.0.1:9200`。 |
| 端口规划 | 默认：前端本地 8080（`npm run serve`）或 4173 等自定义端口，后端 FastAPI 使用 8000。确保本机防火墙放行这些端口。 |
| SSL 域名（可选） | 若需固定二级域名/HTTPS，需要在 cpolar 购买带宽套餐并在控制台创建保留二级域名。基本版使用随机临时域名即可。 |

---

## 2. 后端（FastAPI）本地运行

1. **安装依赖**
   ```bash
   cd anpu-backend
   python3 -m venv venv        # 可选
   source venv/bin/activate    # Windows 使用 venv\Scripts\activate
   pip install --upgrade pip
   pip install -r requirements.txt
   ```

2. **准备配置与数据库**
   - 确保 `anpu-backend/.env` 存在，可沿用开发环境配置，重要参数：`SECRET_KEY`、`DATABASE_URL`、`CORS_ORIGINS`。
   - 如果使用 SQLite，请确认 `anpu.db` 位于后端根目录；若使用 MySQL，保证数据库已初始化。

3. **启动服务**
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```
   打开 `http://127.0.0.1:8000/docs` 确认接口可用。

---

## 3. 前端（Vue 2）本地运行

1. **安装依赖**
   ```bash
   cd anpu-frontend
   npm install
   ```

2. **配置后端地址**
   - `src/api/request.js` 默认读取 `VUE_APP_API_BASE_URL`，在项目根目录创建 `.env.development.local`（或直接在运行命令前设置环境变量）：
     ```
     VUE_APP_API_BASE_URL=http://localhost:8000
     ```
   - 后续通过 cpolar 暴露后端地址后，可再改成公网地址以验证跨域，但本地阶段保持 localhost 以减少延迟。

3. **启动前端开发服务器**
   ```bash
   npm run serve    # 默认 8080 端口
   ```
   浏览器访问 `http://127.0.0.1:8080` 确认页面与接口正常。

---

## 4. 安装并初始化 cpolar

1. **下载安装**
   - Windows：下载 `.exe`，双击安装。
   - macOS：`brew install cpolar/tap/cpolar`。
   - Linux（以 Debian/Ubuntu 为例）：
     ```bash
     wget https://www.cpolar.com/static/downloads/cpolar-stable-linux-amd64.deb
     sudo dpkg -i cpolar-stable-linux-amd64.deb
     ```

2. **登录并绑定 Authtoken**
   ```bash
   cpolar authtoken <从控制台复制的token>
   ```

3. **启动后台服务**
   ```bash
   cpolar service start
   ```
   打开浏览器访问 `http://127.0.0.1:9200`（默认管理界面），登录 cpolar 账号，即可可视化管理隧道。

---

## 5. 创建公网隧道

### 5.1 快速临时域名方式

1. **后端隧道**
   ```bash
   cpolar http 8000
   ```
   终端会显示 `Forwarding` 地址（如 `https://abc123.cpolar.cn`）。将该地址记为 `API_BASE_URL`，用于远程前端调用。

2. **前端隧道**
   ```bash
   cpolar http 8080
   ```
   得到另一个公网地址（如 `https://xyz789.cpolar.cn`）。外部用户直接访问即可看到前端页面，页面中的所有 API 请求仍指向本地 `http://localhost:8000`，所以若需外部访问，应把 `.env` 中的 `VUE_APP_API_BASE_URL` 改成后端公网地址并重新 `npm run serve`。

3. **保持运行**
   - 上述命令会占用当前终端窗口。若需后台运行，可在 cpolar WebUI → `状态` → `在线隧道` 中查看地址并使用“复制命令”生成守护进程。

### 5.2 保留二级域名 + 长期运行（推荐）

1. **在 cpolar 控制台**
   - “云端隧道管理” → “保留二级域名”，分别为前端和后端创建域名（例如 `anpu-frontend.cpolar.cn`、`anpu-backend.cpolar.cn`），选择线路 `中国区 VIP-Lite` 或更高套餐。

2. **编辑本地配置文件**（示例：`~/.cpolar/cpolar.yml`）：
   ```yaml
   tunnels:
     anpu-backend:
       proto: http
       addr: 8000
       domain: anpu-backend.cpolar.cn
     anpu-frontend:
       proto: http
       addr: 8080
       domain: anpu-frontend.cpolar.cn
   ```

3. **重新启动服务**
   ```bash
   cpolar service stop
   cpolar service start
   ```
   WebUI 中可见两个 “在线隧道”，分别对应前后端固定公网地址。

---

## 6. 浏览器访问与验证

1. 使用 `https://anpu-frontend.cpolar.cn`（或临时地址）在外部网络访问前端，登录/操作流程应与本地一致。
2. 观察浏览器开发者工具 Network 面板，确认所有 API 请求指向 `https://anpu-backend.cpolar.cn/api/v1/...` 且状态 200。
3. 若遇到 CORS 报错，需要在 `anpu-backend/.env` 的 `CORS_ORIGINS` 中加入新公网域名，重启后端服务。
4. 上传文件等涉及 `/uploads` 的接口同样走后端公网地址，无需额外配置。

---

## 7. 常见问题与建议

| 问题 | 处理建议 |
| --- | --- |
| 端口被占用 | 修改 `npm run serve` 的端口（`npm run serve -- --port 8090`）或在 `cpolar` 中更换 `addr`。 |
| 隧道掉线 | 确保本机网络稳定，可配置 `cpolar service start --watchdog`、退出休眠模式。 |
| 性能瓶颈 | cpolar 免费线路带宽有限，若用户量较大请升级套餐或考虑云服务器正式部署。 |
| HTTPS 证书 | cpolar 自动为域名配置证书，无需额外操作。自定义域名需按其官方指引做 CNAME 解析。 |
| 多人协作 | 每位开发者可单独开启隧道，或使用 cpolar 的“分组 Token”功能生成独立子账号。 |

---

通过上述步骤，当前的前端与后端即可在无需改动代码的情况下，通过 cpolar 映射到公网，供远程用户在浏览器中直接体验与调试。需要长期稳定访问时，建议仍计划正式服务器部署，以避免内网穿透的带宽和在线时长限制。***
