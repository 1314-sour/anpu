# 安全设置功能说明

## ✅ 已实现的功能

### 1. 修改登录密码

**步骤流程:**
1. 验证身份 - 输入原密码
2. 修改登录密码 - 输入新密码和确认密码
3. 完成 - 显示成功提示

**表单验证:**
- ✅ 原密码不能为空
- ✅ 新密码不能为空
- ✅ 两次密码必须一致
- ✅ 密码长度不少于6位
- ✅ 后端验证原密码是否正确

**API调用:**
```javascript
POST /api/v1/user/security/password
Body: {
  old_password: "原密码",
  new_password: "新密码"
}
```

---

### 2. 修改手机绑定

**步骤流程:**
1. 验证身份 - 显示账户和当前手机号
2. 绑定新手机 - 输入新手机号
3. 完成 - 显示成功提示

**表单验证:**
- ✅ 手机号不能为空
- ✅ 手机号格式验证(1开头的11位数字)
- ✅ 修改成功后立即更新页面显示

**API调用:**
```javascript
PUT /api/v1/user/profile
Body: {
  phone: "新手机号"
}
```

**数据库更新:**
- 更新 `user_profiles` 表的 `phone` 字段
- 自动重新计算安全等级分数

---

**表单验证:**
- ✅ 问题不能为空

## 🎯 安全等级计算规则

**总分100分,分为4个部分,每部分25分:**

1. **密码已设置** - 25分 (固定)
2. **手机已绑定** - 25分
3. **邮箱已绑定** - 25分
4. **密保已设置** - 25分

**等级划分:**
- 分数 < 60: **低** (红色) - "存在风险,建议完善安全设置"
- 分数 60-80: **中** (橙色) - "安全等级一般,建议继续完善"
- 分数 >= 80: **高** (绿色) - "账号非常安全"

**实时更新:**
修改任何安全设置后,安全等级会自动重新计算并更新显示。

## 📊 页面交互效果

### 修改按钮状态

- **未设置**: 显示 "⚠ 未设置 | 设置" (橙色警告图标)
- **已设置**: 显示 "✓ 已设置 | 修改" (绿色成功图标)

### 弹窗样式

- **步骤条**: 使用Element UI的Steps组件,清晰显示当前步骤
- **步骤1**: 验证身份,显示账户信息
- **步骤2**: 输入新信息,带表单验证
- **步骤3**: 成功页面,显示大号绿色对勾

### 按钮控制

- **取消**: 所有步骤都可以取消
- **上一步**: 步骤2可以返回步骤1
- **下一步/提交**: 根据当前步骤显示不同文字
- **完成**: 步骤3显示完成按钮,关闭弹窗

## 🔧 后端API接口

### 1. 获取用户资料
```

Headers: Authorization: Bearer {token}

Response:
{
  "code": 200,
  "data": {
    "username": "admin",
    "phone": "0519-82612300",
    "email": "admin@anpu.com",
    "contact_person": "王翔",
    "address": "江苏省常州市新北区",
    "avatar": "",
    "register_time": "2025-12-05",
    "security_question_set": true
  }
}
```

### 2. 更新用户资料
```
PUT /api/v1/user/profile
Headers: Authorization: Bearer {token}
Body: {
  "phone": "新手机号",              // 可选
  "email": "新邮箱",                // 可选
  "security_question_set": true,   // 可选
  "contact_person": "联系人",      // 可选
  "address": "地址"                // 可选
}

Response:
{
  "code": 200,
  "message": "保存成功"
}
```

### 3. 修改密码
```
POST /api/v1/user/security/password
Headers: Authorization: Bearer {token}
Body: {
  "old_password": "原密码",
  "new_password": "新密码"
}

Response:
{
  "code": 200,
  "message": "密码修改成功"
}

Error Response:
{
  "code": 400,
  "message": "旧密码错误"
}
```
